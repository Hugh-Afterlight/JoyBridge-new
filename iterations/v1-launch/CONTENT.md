# JoyBridge · CONTENT · v1-launch

> 本文件是 JoyBridge 重构版首发内容索引。App 不是网站，所以这里的“内容槽”主要指界面文案、状态文案、诊断字段和朋友测试说明。配合 `PRD.md`、`../../DESIGN.md`、`../../ARCHITECTURE.md` 使用。

## 全局产品文案

- **App 名称** 📝
  > JoyBridge

- **一句话说明** 📝
  > 把手柄按键变成 Mac 键盘动作。

- **版本标识** 📦
  - 数据源：`AppInfo.currentTestVersion` + `AppInfo.releaseDate`
  - 要求：主窗口、菜单栏、诊断信息使用同一来源。

## 主窗口内容槽

### 顶部总状态

- **标题** 📝
  > JoyBridge

- **状态副标题** 🤖
  - 生成需求：根据运行状态生成一句短提示，最多 18 个中文字符。
  - 示例方向：`已准备好`、`需要辅助功能权限`、`等待手柄连接`、`映射已暂停`。

- **复制诊断按钮** 📝
  > 复制诊断信息

### 运行检查区

- **区块标题** 📝
  > 运行检查

- **检查项标签** 📝
  > 辅助功能 / 控制器 / 目标控制器 / 映射输出

- **下一步提示** 🤖
  - 生成需求：每种阻塞状态给一条明确动作，不讲技术细节。
  - 例：缺权限时提示打开系统设置；无手柄时提示连接蓝牙手柄；未锁定时提示锁定当前手柄。

### 权限区

- **区块标题** 📝
  > 权限状态

- **按钮文字** 📝
  > 请求授权/打开设置

- **辅助说明** 🤖
  - 生成需求：解释 Accessibility 是“允许 JoyBridge 发送键盘事件”的系统权限；避免吓人的安全措辞。
  - 必须包含当前 App 路径，帮助排查授权到错误安装位置的问题。

### 手柄区

- **区块标题** 📝
  > 手柄状态

- **按钮文字** 📝
  > 重新检测手柄 / 锁定当前 / 清除目标

- **目标锁定说明** 📝
  > 锁定后，JoyBridge 只响应这个手柄。目标手柄未连接时，不会自动切换到其他手柄。

### 映射区

- **区块标题** 📝
  > 映射配置

- **列/字段标签** 📝
  > 手柄按键 / 启用 / 修饰键 / 键盘按键 / 预览

- **默认映射表** 📦
  - 数据源：`MappingManager.defaultMappings()`
  - 要求：README、UI 默认值、测试说明保持一致。

### 运行控制区

- **区块标题** 📝
  > 映射状态

- **按钮文字** 📝
  > 暂停映射 / 启用映射

- **暂停说明** 📝
  > 暂停后仍会监听手柄和更新最近按键，但不会发送键盘输入。

### 诊断区

- **区块标题** 📝
  > 诊断信息

- **复制成功提示** 📝
  > 已复制

- **诊断字段** 📦
  - 数据源：`DiagnosticReport`
  - 字段：App 版本、macOS 版本、App 路径、Accessibility 状态、当前手柄、目标手柄、最近按键、暂停状态、映射数量、最近输出结果。

## 菜单栏内容槽

- **菜单状态标题** 🤖
  - 生成需求：短状态，适合菜单栏显示。
  - 示例：`JoyBridge 运行`、`JoyBridge 未授权`、`JoyBridge 暂停`。

- **菜单项** 📝
  > 启用映射 / 暂停映射 / 显示 JoyBridge / 重新检测手柄 / 重新检测权限 / 请求授权/打开设置 / 复制诊断信息 / 退出 JoyBridge

## 朋友测试文案

- **测试指南来源** 📦
  - 数据源：`FRIEND_TESTING.md`
  - 要求：重构后如果 UI 文案或流程变化，同步更新朋友测试步骤。

- **反馈模板字段** 📦
  - 数据源：诊断信息 + 测试者描述。
  - 字段：设备型号、macOS 版本、手柄型号、卡在哪一步、诊断文本、是否能复现。

## 文案规则

- 使用中文界面文案，英文技术名词只在必要时保留，例如 Accessibility。
- 按钮文字短、直接、动词开头。
- 不使用游戏化语气，不夸张承诺“支持所有手柄”。
- 遇到系统能力限制时说清原因和下一步，而不是让用户猜。
